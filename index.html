JavaScript Interview Questions

What is the difference between let, const, and var?

var: Function-scoped, can be redeclared, and initialized with undefined if not initialized.

let: Block-scoped, cannot be redeclared in the same scope, but can be reassigned.

const: Block-scoped, cannot be redeclared or reassigned, and must be initialized at declaration.

What is a closure in JavaScript?

A closure is a function that has access to its outer (enclosing) function's variables even after the outer function has returned. This allows for data privacy and state preservation in JavaScript.

Example:

function outer(x) {
  return function inner(y) {
    return x + y;
  };
}
const add5 = outer(5);
console.log(add5(3)); // Output: 8

What are promises in JavaScript?

Promises are objects representing the eventual completion or failure of an asynchronous operation. They have three states:

  • Pending: Initial state, not fulfilled or rejected
  • Fulfilled: Operation completed successfully
  • Rejected: Operation failed

Promises can be chained using .then() and .catch() methods.

Explain the 'this' keyword in JavaScript.

The 'this' keyword refers to the object it belongs to, depending on the execution context:

  • Global context: Refers to the global object (window in browsers)
  • Function context: Depends on how the function is called
  • Method context: Refers to the object the method belongs to
  • Constructor context: Refers to the newly created instance

Arrow functions don't have their own 'this'; they inherit it from the parent scope.

What is the difference between == and ===?

== (Loose Equality): Compares values after converting them to a common type (type coercion).

=== (Strict Equality): Compares both value and type without conversion.

Example:

5 == '5'  // true
5 === '5' // false

What is event bubbling and event capturing?

Event Capturing: The event starts from the window and moves down to the target element.

Event Bubbling: The event starts from the target element and bubbles up to the window.

You can control the phase using addEventListener's third parameter: true for capturing, false for bubbling (default).
